Once your profile is complete, navigate to the API section in your dashboard:

  • Click on Generate API Key

Deploying Your Portfolio

provides a prebuilt front-end template that you can deploy as a Docker container. Follow the steps below to get your portfolio online:

  1. Install Docker:

    Ensure that you have Docker installed on your system. You can download it from Docker's official website

  2. Pull the Profile Host Portfolio Docker Image

    Run the following command in your terminal:

    docker pull {{imageName}}/portfolio:tag-name
  3. Deploy the Docker Container

    Use the following command to run the portfolio container:

    docker run -d -p 8080:80 -e {{apiKeyName}}=your_api_key {{imageName}}/portfolio:tag-name

    Replace `your_api_key` with the API key you generated earlier. Only the classic tag is available on free tier

  4. Access Your Portfolio

    Once the container is running, open your browser and go to:

    http://localhost:8080

    If you are deploying on a cloud service or a remote server, replace `localhost` with the appropriate domain or IP address.

Sharing Your Portfolio

After deploying your portfolio, you can share the link to your deployed container (e.g., http://yourserver.com:8080 ) with potential clients or employers.

Using the API

For those who wish to develop their own front-end instead of using our Docker image, provides a GraphQL API to fetch profile data securely. You must include your API key in the request headers to authenticate. Refer to the API Section for a guide.

Updating Your Profile

If you update any details in your Profile Host account, the changes will be reflected on your portfolio site automatically. Simply refresh your portfolio page to see the updated information.

Troubleshooting

  1. API Key Not Working
    • Ensure that you copied the correct API key.
    • Check that the API key is correctly set in the docker run command.
  2. Portfolio Not Loading
    • Verify that Docker is running.
    • Check that the container is running using docker ps.
    • Ensure port 8080 is not blocked or used by another service.

Updating to the Latest Version

To update your portfolio, pull the latest Docker image and restart the container:

docker pull {{imageName}}/portfolio:tag-name
docker stop $(docker ps -q --filter ancestor={{imageName}}/portfolio:tag-name)
docker run -d -p 8080:80 -e {{apiKeyName}}=your_api_key {{imageName}}/portfolio:tag-name
